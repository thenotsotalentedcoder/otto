# Product

## Register

product

## Users

Solo technical user (CS student / AI engineer). Constantly context-switching between multiple projects. Captures ideas on phone and laptop. Thinks in mixed Urdu/English. Wants zero organizational overhead. Uses the app many times a day — capture is the primary motion, retrieval is the secondary.

## Product Purpose

A personal second brain that lives in your daily flow. One chat thread where you dump anything (ideas, links, voice notes, images, code snippets, decisions) and retrieve it all through natural conversation. The AI has full context of everything ever saved. No folders, no tags the user manages, no friction.

## Brand Personality

Warm, intimate, conversational. Feels like a personal companion that knows you — not a generic tool. Like texting a version of yourself that remembers everything. The interface should feel like it belongs to the user, not like a product they're borrowing.

## Anti-references

- **Notion**: structured, database-heavy, organizational. This app has no structure the user manages.
- **Generic AI chat UI (ChatGPT/Claude)**: centered chat column with no personality, no identity. This should feel deeply personal.
- **Obsidian**: dark, graph-heavy, plugin-ecosystem complexity. The opposite of zero-friction.
- **Linear**: clean but corporate SaaS B2B polish. Cold, team-product feel.

## Design Principles

1. **The thread is the product.** The infinite chat is not a feature — it is the whole experience. Every design decision serves that thread.
2. **Capture is sacred.** Anything that adds friction to sending a message is a failure. Input must always be ready, always fast.
3. **AI is a quiet presence.** In dump mode, AI acknowledgments should not interrupt the flow. They whisper, they don't announce.
4. **Nothing competes equally.** Strong visual hierarchy everywhere. User messages are content; AI acks are system; AI responses are considered replies. Each has a different weight.
5. **Personal, not polished.** The app should feel like it was built for one person. Not generic, not sanitized.

## Accessibility & Inclusion

WCAG AA minimum. Mixed Urdu/English content (RTL-aware text handling needed eventually). Voice input is a first-class feature — offline fallback matters. Touch targets minimum 44px for mobile use.
